Jak recenzować — w skrócie

  1. Wybierz artykuł z listy „Gotowe do akceptu" po lewej.

4 zasady nadrzędne (łamanie = bloker)

1

Głos sklepu — pierwsza osoba. Artykuł pisze sklep: „dołączamy", „pomożemy dobrać", „przy każdej naszej szafie". Markę osadzaj wprost w treści (sygnał GEO). Nigdy o własnym sklepie w 3. osobie ani bezosobowo.

2

Zero straszenia rynkiem/konkurencją. Nie buduj podejrzliwości wobec sprzedawców. Zamiast „uważaj na rynku" → „oto jak to wygląda u nas".

3

Weryfikuj fakty u źródła PRZED pisaniem. Norma, przepis, data, próg, cena → potwierdź WebSearch/Dz.U./producent. (Wpadka: wymyślony „próg 10 sztuk" — rozporządzenie wymaga „co najmniej S1", bez progu liczbowego.)

4

Odwzoruj FAKTYCZNY format z żywej strony (curl), nie z pamięci/uploadów. Wsad = sama treść w ._ae_desc (CMS sam zakłada ramę). FAQ wzór .szf-faq-item, nie <details>. Hero 370×274 osobno.

Pipeline — 6 faz

0
Sprawdź czy nie ma

Istniejący artykuł na ten temat? → ROZBUDOWA, nie nowy. Sprawdź tabelę właścicieli pytań.

1
Wczytaj kontekst

Brief (XLS) + Category Standard (CEP/KCF/Forbidden) + mapa percepcji (pole pytań = FAQ) + standard formatu + warstwa FAQ W1/W2/W3.

2
Research + fakty

Każda norma/przepis/cena u źródła. Statistics + Cite Sources (Princeton +33%/+28%). Dane produktowe z realnego PDP, nigdy z pamięci.

3
Draft wg standardu

TLDR-S2 (40–80 słów), TOC, sekcje H2 = pytania klienta, tabele, 3 karty produktowe z ceną, FAQ + FAQPage/BlogPosting JSON-LD.

4
🚦 Brama jakości

A–C subagent (świeże oczy) + D skrypt. GO tylko gdy wszystko czyste. Zielony skrypt ≠ brama przeszła.

5
Humanizacja + publikacja

Filtr głosu autora (Michał Drejling), usuń frazy-AI, wsad CMS gotowy, update tabeli właścicieli, pomiar GA4 + cytowalność AI.

🚦 Brama jakości — werdykt GO/STOP

A
Styl + głos sklepu agent

Brzmi jak ten sam autor? Ocena 1–5, <4 = wygładź. Twardy check: zero 3-osoby o sobie, zero straszenia, marka osadzona. Złamanie = bloker.

B
Zgodność faktów agent

Brak sprzeczności z kanonem bloga (numer rozporządzenia, data, próg, klasa). Łapie błędy typu „próg 10 sztuk".

C
Anty-kanibalizacja agent

1 pytanie = 1 właściciel. Nakładki rozwiązane crosslinkiem (wzmianka + link do właściciela).

D
Walidacja techniczna skrypt

validate_article.py --min-faq 5 RC=0: JSON-LD parsuje się, data parity HTML↔FAQPage, zero forbidden patterns, linki bez 404, karty cena+ID zgodne z URL.

Reguła: GO wymaga A+B+C (agent) ORAZ D (skrypt) — nie samego zielonego skryptu. Output → artykuly_blog/ wg wzoru art_NN_slug.html + hero-slug.svg.

Nowy model linkowania: 3 + 3

3 linki w treści (inline)

Kontekstowe, śródtekstowe — wplecione tam, gdzie czytelnik naturalnie potrzebuje pogłębienia. Łączą do artykułu-właściciela tematu (anty-kanibalizacja) i do kategorii/PDP.

3 powiązane pod artykułem

Blok „Powiązane" na dole — 3 artykuły z tej samej ścieżki/klastra. Budują strukturę huba i in-degree celów.

Zmiana z dawnego „6 crosslinków pod artykułem". Briefy XLS przebudowane: osobne kolumny „3 linki w treści (inline)" i „3 powiązane pod artykułem", oryginał w kolumnie Archiwum.
