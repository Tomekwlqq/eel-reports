WF-03
Anomalia Alert
🔴 EVENT — spike / drop >15%
OutputANOMALY_BRIEF.md
Trigger
Auto z WF-02 gdy delta CR lub revenue >15%

lub manualny gdy klient zgłasza problem
Agenci / skille
cro-analyst (diagnoza: gdzie pęknięcie)
ga4-puller (segment: device, channel, URL)
strateg (czy to symptom czy przyczyna)
Co robi
Segmentuje anomalię: urządzenie / kanał / strona / czas. Szuka przyczyny (nowy landing? zmiana ceny? problem tech?). Hierarchia STRONA→SHOPPER→AKCJE→DANE.
Rezultat
ANOMALY_BRIEF.md

· co się stało
· gdzie (URL / segment)
· hipoteza przyczyny
· trigger WF-04 na URL

Fast-lane. Gdy coś się sypie, nie czekasz do poniedziałku. Brief idzie do Tomka + do klienta w ciągu godziny od wykrycia.

Truth Checker

CO WIEM
Klient ma Cowork Team + GA4. Clarity i dane transakcyjne to założenie. Masz działające EEL pipeline (capture, vision, CS), strateg + agentów. Claude Code obsługuje agents/skills/workflows.

CO ZAKŁADAM
Clarity API jest dostępne i klient da token. GA4 MCP działa lub masz oauth. Dane transakcyjne = Shopify/inny silnik ma API. Klient jest na Claude Team (nie solo).

CO MOŻE BYĆ BŁĘDNE
Clarity może nie mieć publicznego API (Clarity eksportuje tylko przez dashboard). Cowork Team może nie obsługiwać custom agentów (tylko skille przez plugin). Klient może mieć inny silnik niż Shopify.

CO ZWERYFIKOWAĆ
Clarity API dostępność · format danych transakcyjnych klienta · czy Cowork Team pozwala na custom agents czy tylko skille · jakie URL-e są priorytetem dla klienta.

CO POPRAWIĘ Z JEDNYM INPUTEM
Gdy wiem czy Clarity ma API — zmienię WF-02/03 albo wytnę tę gałąź. Gdy wiem jak Cowork distribuuje narzędzia — zmienię typy artefaktów (agent→plugin).
