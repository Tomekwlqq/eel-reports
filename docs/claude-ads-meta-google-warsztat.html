Bliźniak podręcznika SEO/GEO. Zakres pracy płatnej reklamy robionej Claudem + MCP: research, struktura, kreacja, launch, optymalizacja — językiem naturalnym zamiast klikania w Ads Managerze. 1Meta vs Google — dwie maszyny, jeden warsztat

Obie to płatny ruch, ale łapią klienta w innym momencie. Mylenie ich = przepalony budżet. Claude+MCP pozwala obsłużyć obie z terminala, językiem naturalnym.

META (FB/IG)

Demand GENERATION — przerywasz scroll

Klient nie szuka — Ty go zaczepiasz kreacją. Silnik to AI (Andromeda): targeting i tak robi maszyna, Twoja dźwignia #1 to różnorodność kreacji. Pytanie klienta: „czemu przepalam budżet i nie wiem które reklamy działają".

GOOGLE

Demand CAPTURE — łapiesz intencję

Klient już szuka („sejf domowy cena") — Ty stajesz na drodze. Dźwignia: keyword/intent match, struktura konta, Quality Score, GAQL do diagnozy. Pytanie klienta: „płacę za kliki które nie kupują".

Dlaczego Claude + MCP teraz

Ads Manager to klikanie. MCP daje to samo przez język: „pokaż kampanie z ROAS < 1 z 7 dni i zapauzuj zombie" = jedna komenda zamiast 20 kliknięć. Claude czyta insights, łączy z GA4, proponuje ruch — Ty zatwierdzasz. To jest warsztat: analiza + decyzja, nie ręczna obsługa panelu.

2Pipeline kampanii — to JEST warsztat

Jak biegnie zlecenie od briefu do optymalizacji. Każdy krok ma narzędzie, którym to robisz.

0

Brief + dostępy

Cel (sprzedaż/leady/zasięg), budżet, konto Meta (System User token / OAuth) + Google Ads (customer ID, GAQL). GA4 jako prawda o konwersji. Bez GA4 mierzysz kliki, nie przychód.

1

Research odbiorcy / intencji

Meta: MCP search_interests / search_behaviors / search_demographics + estimate_audience_size. Google: Senuto/keyword + GAQL keyword_view. Łączysz z mózgiem (ctx_search).

2

Struktura konta

Kampania → ad sety → ads. Claude proponuje strukturę pod cel (Advantage+ vs manualne). MCP: create_campaign / create_adset — zawsze startują PAUSED (safety default).

3

Kreacja

Brief kreacji wg wzorca creative-director-ai (Audience / Pain / Promise / Proof / CTA). Meta: różnorodność = dźwignia #1 (Andromeda karze podobieństwo >60%). Generacja przez /ads creative + 4 agenty kreatywne. MCP upload_ad_image / create_ad_creative.

4

Launch kontrolowany

Reklamy włączasz świadomie (PAUSED→ACTIVE) po sanity-checku. CAPI/Pixel health zweryfikowane (ads-meta 50 checks) — bez tego Meta optymalizuje na ślepo.

5

Analiza wyników

Meta: get_insights (spend/ROAS/CPA, breakdowny: age/placement/device, creative-asset). Google: GAQL FROM campaign / ad_group_ad. Claude liczy anomalie (±σ), wykrywa zombie ads, łączy z GA4.

6

Optymalizacja regułowa

„ROAS < 0.5 w 7 dniach → pauza", „skaluj budżet +20% gdy ROAS > cel". MCP write: update_campaign / update_adset (budżet, status). Każda zmiana na żywym koncie klienta = Ty zatwierdzasz.

Granica MCP V1: zarządza strukturą i budżetem, nie tworzy kreacji od zera. Ad fatigue / refresh = osobny pipeline kreatywny (/ads creative + creative-director).

3MCP — Twój silnik (jak Senuto w SEO)

Masz podpięte oba: Meta Ads MCP (read+write) i Google Ads MCP (GAQL). To różnica między „doradzam" a „robię z terminala".

PlatformaRead (raporty)Write (zarządzanie)
Meta Ads MCPad accounts, campaigns/adsets/ads, get_insights (spend/ROAS/CPA, breakdowny), audience size, anomaly/zombiepauza, budżet, rename, create_* (PAUSED), audiences/lookalike, CAPI upload_conversion_events
Google Ads MCPGAQL search: campaign, ad_group_ad, keyword_view, metrics. list_accessible_customers(V1 read-heavy — zmiany planuj, wykonuj świadomie)

GAQL — gotowiec z mózgu (concepts/gaql)

-- Kampanie + metryki 30 dni
SELECT campaign.name, campaign.status, metrics.impressions, metrics.clicks,
  metrics.cost_micros, metrics.conversions, metrics.conversions_value
FROM campaign
WHERE segments.date DURING LAST_30_DAYS AND campaign.status = 'ENABLED'
ORDER BY metrics.cost_micros DESC

⚠️ Pułapki (z mózgu + memory — oszczędzą Ci godzin)

  • Token Meta: System User token (Graph API) ≠ OAuth dla mcp.facebook.com — to osobne flow. Stan na dziś: token GLOV zwraca „API access blocked" — wymaga odświeżenia przed live runem.
  • asset_feed_spec bez zawężenia → „too much data" → używaj asset_feed_spec{link_urls}.
  • limit=100 z creative fields → błąd → limit=25 (paginacja sama działa).
  • purchase_roas jest w insights — przychód = spend × ROAS, nie pobieraj osobno.
  • 365-dniowe queries = wolne i drogie tokenowo — tnij okna.

4Warsztat A: Meta (FB/IG)

Po Andromeda (Meta AI engine, paź 2025) gra się zmieniła: targeting robi maszyna, Ty wygrywasz kreacją i czystością danych.

Creative diversity = dźwignia #1

Andromeda filtruje ads 10 000× bardziej złożonym modelem. Reklamy o podobieństwie >60% są tłumione. Wniosek: wiele różnych kątów kreatywnych > jedna „idealna" reklama.

Pixel / CAPI health

Jeśli zdarzenia konwersji nie wpływają czysto (Pixel + CAPI), Meta optymalizuje na ślepo. To pierwszy audyt — ads-meta sprawdza 50 punktów. CAPI: upload_conversion_events (PII hashowane).

Co robisz Claudem (Meta)

  • Audyt 50 checks (ads-meta): Pixel/CAPI, creative diversity & fatigue, struktura, targeting, Advantage+.
  • Research odbiorcy: search_interests/behaviors/demographics + estimate_audience_size — bez zgadywania.
  • Diagnoza wydajności: get_insights z breakdownami (placement, device, age, creative-asset — która grafika/nagłówek ciągnie).
  • Optymalizacja: reguły pauza/budżet przez update_*, nowe kampanie jako PAUSED.

5Warsztat B: Google

Tu wygrywa intencja i struktura. Claude + GAQL = diagnoza konta w sekundy zamiast grzebania w panelu.

Co robisz Claudem (Google)

  • Audyt konta (/ads google): struktura kampanii, Quality Score, marnotrawstwo budżetu, search terms.
  • GAQL diagnostyka: kampanie po koszcie, kreacje z final_urls, keyword_view z match_type — co zżera budżet bez konwersji.
  • Keyword/intent: Senuto (PL/CEE) + GAQL — mapa fraz pod intencję transakcyjną.
  • Most do GEO/SEO: te same frazy intencyjne karmią podręcznik SEO/GEO — paid waliduje, organic skaluje za darmo.
Synergia z poprzednim podręcznikiem

Google Ads search terms = realne zapytania, za które ktoś płaci. Wrzuć je do GEO/FAQ (Answer Equity, 1500–1750 słów) — i przestajesz płacić za ruch, który zacznie przychodzić organicznie. To jeden klient, dwa podręczniki.

6Procedury gotowe (recipes — odpalaj jak przepis)

Sedno podręcznika: powtarzalne sekwencje. Każda procedura = kroki z dokładnym narzędziem. Kopiuj i wykonuj.

PROC-1 · Audyt konta nowego klienta (Meta + Google)

Kiedy: onboarding, pierwszy kontakt. Cel: raport „co jest zepsute + gdzie ucieka budżet".

  1. ctx_search(["meta ads audyt", "gaql diagnoza"]) — najpierw mózg.
  2. /ads audit meta → 50 checks (Pixel/CAPI, creative diversity, struktura, Advantage+).
  3. /ads audit google → struktura, Quality Score, marnotrawstwo, search terms.
  4. Meta MCP get_insights(level=campaign, time_range=last_30d) + breakdown placement/device.
  5. Google MCP GAQL FROM campaign ORDER BY cost_micros DESC.
  6. Złóż raport HTML (wzorzec mega-audytu) → publish.

PROC-2 · „Gdzie ucieka budżet" (diagnoza marnotrawstwa)

  1. Google: GAQL FROM keyword_view + search_term_view → frazy z wydatkiem i 0 konwersji → lista negatywów.
  2. Meta: get_insights(breakdown=ad_format_asset / image_asset) → które kreacje palą bez ROAS = zombie.
  3. Cross z GA4 (run_report) → czy kliki w ogóle dochodzą do koszyka.
  4. Output: lista „zatnij to" + szacowana oszczędność PLN/mies.

PROC-3 · Launch kampanii Meta od zera

  1. Research odbiorcy: search_interests / search_behaviors + estimate_audience_size.
  2. Struktura: create_campaign → create_adset (startują PAUSED).
  3. Kreacja: brief wg creative-director-ai (Audience/Pain/Promise/Proof/CTA) → /ads creative → min. 3–5 różnych kątów (Andromeda karze podobieństwo).
  4. upload_ad_image → create_ad_creative → create_ad.
  5. Sanity: Pixel/CAPI health (ads-meta) zanim włączysz.
  6. update_adset(status=ACTIVE) — świadomy launch po sanity-checku.

PROC-4 · Optymalizacja tygodniowa (regułowa)

  1. get_insights(time_range=last_7d) Meta + GAQL 7d Google.
  2. Reguły: ROAS < 0.5 (7d) → kandydat do pauzy · ROAS > cel → skala budżet +20%.
  3. Przedstaw Tomkowi listę zmian → ZATWIERDZENIE (żywe konto = realne pieniądze).
  4. Po OK: update_campaign / update_adset (status, budżet).
  5. Wpis @log do historii klienta (co zmieniono i czemu).

PROC-5 · Creative refresh (ad fatigue)

  1. get_insights(breakdown=creative) + frequency → wykryj zmęczone kreacje (CTR spada, freq rośnie).
  2. /ads creative + 4 agenty kreatywne → nowe kąty z briefu.
  3. Podmiana jako nowe ads (PAUSED) → A/B ze starymi → wygrane skaluj.

PROC-6 · Raport tygodniowy klientowi

  1. Meta get_insights + Google GAQL + GA4 run_report (przychód = prawda).
  2. WoW delta (spend/ROAS/CPA/konwersje) + anomalie ±σ.
  3. HTML (sparklines) → publish → notka Discord. Reuse: metoda dashboardu (memory: reference_meta_ads_dashboard).

Każda procedura zaczyna się od ctx_search (mózg) i — gdzie dotyka żywego konta — kończy zatwierdzeniem przez Tomka. To nie autonomia, to warsztat z bezpiecznikiem.

Truth Checker

Co wiem

  • Masz oba MCP (Meta read+write, Google GAQL), skill /ads (250+ checks) + ads-meta, GA4, realne konto GLOV. Wiki ma facebook-ads-mcp/gaql/creative-director.
  • Token Meta dziś NIE działa („API access blocked", OAuthException 200) — potwierdzone live.
  • Andromeda/creative-diversity i pułapki API to wiedza z mózgu, nie zgadywanie.

Co zakładam

  • Że odświeżenie tokenu Meta to formalność (re-auth), nie martwy connector. Nie zweryfikowałem.
  • Że masz dostęp write do kont klientów — zarządzanie regułowe (pauza/budżet) wymaga ads_management, nie tylko read.
  • Że „klienci chcą" reklamy tak jak SEO/GEO — nie znam liczby ani budżetów.

Co może być błędne / ryzykowne

  • Write na żywym koncie klienta = realne pieniądze. Automatyczna pauza/budżet przez MCP to ryzyko — zła reguła = zatrzymana sprzedaż albo przepalony budżet. Każda zmiana musi przejść przez Twoje zatwierdzenie, nie autonomię.
  • Andromeda (paź 2025) i „similarity >60%" to claim ze skilla — zweryfikuj na realnych danych GLOV zanim sprzedasz jako pewnik.
  • MCP V1 nie tworzy kreacji — jeśli problem klienta to ad fatigue, sam MCP nie wystarczy.

Co zweryfikować zanim ruszysz

  • Odśwież token Meta i odpal get_insights na GLOV — potwierdź że dane lecą.
  • Sprawdź zakres tokenu (ads_read vs ads_management) — bez management nie zarządzasz.
  • 1 realny case GLOV (przed/po) zanim pakietyzujesz.

Co poprawiłbym z jednym inputem

Daj odświeżony token Meta + nazwę konta GLOV — odpalę live get_campaigns + get_insights, wstawię realne liczby (ROAS, CPA, zombie ads) do tego podręcznika i zrobię z sekcji 8 pitch oparty na faktach, nie szablonie.
