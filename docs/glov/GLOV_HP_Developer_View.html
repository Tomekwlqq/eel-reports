Krytyczne
GAP-01 Countdown timer w Announcement Bar
Mapping opisuje announcement bar tylko jako "Promo bar — tekst + link". W rzeczywistości na GLOV sekcja zawiera dwa elementy: (1) ticker + (2) oddzielna aplikacja countdown timer "Prezenty dla MAMY 25% taniej!" z odliczaniem (02:12:35:55 — dni/godziny/minuty/sekundy) i kodem promo.

Nowe selektory: [class*="essential_countdown_annoucement_bar"] [class*="countdown_annoucement_bar_wrapper"]
Aplikacja: Colibrius Essential Countdown
GAP-08 Podwójne instancje analytics
Hotjar × 2 (ID: 5026693 + 2026668) · Clarity × 2 (scripts.clarity.ms + www.clarity.ms/tag) · Cookiebot × 2 (direct + via GTM).

Efekt: zdublowane eventy heatmap/session recording, potencjalnie zawyżone dane, zbędny load. To błąd konfiguracji — wymaga natychmiastowej naprawy przez klienta.

Mapping HP powinien mieć warstwę analytics stack check (script count per tool).
Ważne
GAP-02 Sliderule — scrollujący pasek kategorii
sliderule__wrapper zawiera DIV.sliderow z poziomymi linkami kategorii. Scan B zwrócił 99 elementów z [class*="slider"] — większość to sliderule, nie slider produktów. Obecny mapping nie odróżnia tych dwóch typów.

Selektory: .sliderule__wrapper .sliderow a.sliderow__title
GAP-03 Colibrius Essential Products Slider
Sekcje Bestsellery i Nowości są opisane w mappingu jako [product-listing] / DIV.product-listing. Faktycznie carousel renderowany jest przez Colibrius (Glide.js) z własnymi klasami. Przyciski "Dobierz" (javascript:void(0)) to trigger rekomendatora Colibrius inline — nie standardowe linki.

Selektory: [class*="essential_product_slider"] .glide--carousel
GAP-04 Tolstoy Video Carousel — CSS Modules hash
Klasy Tolstoy zawierają React CSS Modules hash: _spotlightVideosContainer_zvvi2_235 — niestabilne, zmienią się przy każdym deployu. Mapping opisuje sekcję jako [app-wrapper / Spotlight] bez selektora.

Stabilny selektor: .tolstoy-carousel [class*="tolstoy-carousel"] — używaj tego, NIE hashy.
GAP-05 Growave Wishlist Tab — floating element
Snapshot ujawnia button "Growave wishlist tab" — boczna floating zakładka wishlisty na każdej stronie. Jednak window.Growave = false i brak klasy [class*="growave"]. Mapping opisuje wishlist jako ikonę w headerze — to inny element.

Selektor: button[aria-label*="Growave"] lub button[aria-label*="wishlist"] (wymaga weryfikacji dokładnego aria-label)
GAP-06 SSW Instagram Feed — dostawca inny niż opisany
Klasy: ssw-instagram-content, ssw-share-faves, ssw-icon-star. Script: cdn.nfcube.com/instafeed-*.js — może być NF Cube Instafeed, nie Social Snowball. Mapping opisuje jako [app-wrapper] "embed social media" bez identyfikacji dostawcy.

Selektory: [class*="ssw-instagram"] .ssw-instagram-content
GAP-07 Trusted Shops Badge — Shadow DOM
Script załadowany, w snapshrocie widoczny button "Znak jakości Trusted Shops Ochrona Kupującego 4,65 Doskonały". Jednak [class*="trustedshops"] i [class*="tse"] zwracają null — badge renderowany w Shadow DOM lub iframe, niedostępny przez standardowy CSS selector.

Detekcja: script[src*="trustedshops"] (przez script src, nie DOM)
Informacyjne
GAP-09 GetResponse jako Email + Chat (nie Klaviyo)
GLOV używa GetResponse (i.gr.chat + ga.getresponse.com) zamiast Klaviyo (standard Shopify). window._learnq = undefined. elements_dictionary.json / platform_patterns.json mogą zakładać Klaviyo dla sklepów Shopify — co jest nieprawidłowe dla GLOV.
GAP-10 Onet DlApi — PL-specyficzne narzędzie reklamowe
lib.onet.pl/s.csr/build/dlApi/minit.boot.min.js — Onet Display API, polskie RTB/display advertising. Brak kategorii "PL-specific ad networks" w platform_patterns.json. Pierwsze wykrycie — dodać jako nową kategorię dla rynku PL.
